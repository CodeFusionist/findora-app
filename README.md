# 📦 Findora — Lost Item Tracking & Claim Management System

---

## 📌 Overview

Findora is a mobile-based lost item tracking and claim management system designed to help users report, discover, and reclaim lost items efficiently.

Originally developed as a web-based system, Findora has evolved into a **Flutter-powered mobile application** backed by **Supabase**, enabling real-time interactions, secure authentication, and scalable data handling.

---

## 🚀 Evolution of the System

- 🖥 Started as a web-based Lost & Found system (academic project)  
- 📱 Expanded into a mobile application using Flutter  
- 🌍 Transitioned from a localized solution (Kibwezi) to a scalable national concept  
- ⚙️ Integrated real-time features and modern backend architecture  

This evolution reflects a shift from a simple system to a **production-ready application design**.

---

## ❗ The Problem

Losing valuable items creates stress and inefficiency due to:

- Lack of a centralized reporting system  
- Poor communication between finders and owners  
- No structured claim verification process  
- High chances of fraud or false claims  

Existing solutions are often informal, fragmented, or limited in scope.

---

## 💡 The Solution

Findora provides a structured digital platform where users can:

- Report lost or found items  
- Browse listings in real time  
- Submit claims with verification details  
- Receive updates on claim status  

The system ensures transparency, traceability, and improved chances of item recovery.

---

## ⚙️ Key Features

- 📲 Mobile-first experience (Flutter app)  
- 🔐 Secure authentication (Supabase Auth)  
- 📦 Lost & found item reporting  
- 📑 Claim submission with verification details  
- 🔄 Real-time claim status updates  
- 🔔 Notification system for users  
- 👤 Guest browsing mode (no login required)  
- 🖼 Image upload for items and claims  

---

## 🏗 System Architecture

Findora is built using a modern, scalable architecture:

- **Frontend:** Flutter  
- **Backend:** Supabase (PostgreSQL + APIs)  
- **Authentication:** Supabase Auth  
- **Realtime:** Supabase Realtime Channels  
- **Storage:** Supabase Storage (images & files)  

### 🔁 Core Flow

1. User reports a lost/found item  
2. Item is stored and displayed in listings  
3. Another user submits a claim  
4. Claim is reviewed and approved/rejected  
5. Both parties receive real-time updates  

---

## 🔐 Key System Design Highlights

- Role-based interaction (poster vs claimant)  
- Duplicate claim prevention  
- Real-time notifications for system events  
- Structured claim verification workflow  
- Guest mode to increase accessibility
